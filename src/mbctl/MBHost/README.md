# Man8S Host

一个Man8S容器的托管主机。
Host实现容器的基本动作方法。

MBHost是一个很大的类，因此我们将它拆成多个文件分块实现。
目前我们仅支持在本机执行MBHost，未来会考虑将其扩展到远程执行。

## 软件配置及Host配置

Man8S 的配置应该是针对每个Host采用不同的配置的。不过没有关系，未来会扩展。

mbctl执行命令会在host中读取对应配置并针对host调整正确的参数。

## DNS

Man8S 的域名解析目前还是一个难题，如何构建容器编排系统中的高可靠低延迟热更新的DNS依然是设计的重点问题。

## 命名问题

不能在同一个MBHost中创建同名的容器，否则网络插件无法正常工作。

## 安装CNI插件及CNI配置

Host需要自己安装CNI插件，包括yggoverlay，并自己完成CNI配置。
这方面的参考配置可以在插件的install payload中找到。
